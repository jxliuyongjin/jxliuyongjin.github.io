# 2、样式冲突scoped

默认样式是作用在全局的，某个组件的样式会影响其他组件的样式。避免这种影响，可以在样式标签里生命scoped

<style scoped>
//这里的样式只作用于当前组件，不会影响到其他组件
</stype>

# scoped:原理

  1. 给当前组件模板的所有元素，都会添加一个自定义属性 data-v-hash值
  2. css选择器都被添加[data-v-hash值]的属性选择器
  3. 最终效果就是必须是当前组件的元素，才会有这个自定义属性，才会被这个样式作用到

# 父组件代码
<template>
  <div class="app" style="border: 3px solid #000; margin: 10px">
    我是APP组件
    <!-- 2.父组件对子组件的消息进行监听 -->
    <Son :title="myTitle" @changTitle="handleChange"></Son>
    //同样可以通过.async来省略到书写监听，前提是属性名不是“vulue"
    //<Son :title.async="myTitle"></Son>

<script>
import Son from './components/Son.vue'
export default {
  name: 'App',
  data() {
    return {
      myTitle: '学前端，就来黑马程序员',
    }
  },
  components: {
    Son,
  },
  methods: {
    // 3.提供处理函数，提供逻辑
    handleChange(newTitle) {
      this.myTitle = newTitle
    },
  },
}
</script>
# 子组件代码
<template>
  <div class="son" style="border: 3px solid #000; margin: 10px">
    我是Son组件 {{ title }}

<script>
export default {
  name: 'Son-Child',
  props: ['title'], 
  methods: {

    },
  },
}
</script>

# 4.2、非父子关系

​ eventbus 和 provide & inject

# 4.2.1、eventbus

​ 通过 const Bus = new Vue()，创建一个中间键，提供事件管理和分发，代码如下：

第一步：创建EventBus.js

import Vue from 'vue'

const Bus  =  new Vue()

export default Bus

第二步：a组件监听消息

<script>
import Bus from '../utils/EventBus'
export default {
  data() {
    return {
      msg: '',
    }
  },
  created() {
    Bus.$on('sendMsg', (msg) => {
      // console.log(msg)
      this.msg = msg
    })
  },
}
</script>

第三步：b组件发送消息消息， 发出的消息可以被多个组件监听

<script>
import Bus from '../utils/EventBus'
export default {
  methods: {
    sendMsgFn() {
      Bus.$emit('sendMsg', '今天天气不错，适合旅游')
    },
  },
}
</script>
# 4.2.2、provide & inject

作用：跨层级共享数据，父组件中provide提供的数据，其所有后代组件都可以通过inject拿到

/////////////////////父组件代码：
<script>
import SonA from './components/SonA.vue'
import SonB from './components/SonB.vue'
export default {
  provide() {
    return {
      // 简单类型 是非响应式的
      color: this.color,
      // 复杂类型 是响应式的
      userInfo: this.userInfo,
    }
  },
  data() {
    return {
      color: 'pink', //简单类型是非响应式的，数据修改，其子孙并不会立马显示刷新
      userInfo: { //复杂类型是相应式的
        name: 'zs',
        age: 18,
      },
    }
  },
  methods: {
    change() {
      this.color = 'red'
      this.userInfo.name = 'ls'
    },
  },
  components: {
    SonA,
    SonB,
  },
}
</script> 
/////////////////////子组件代码
<script>
import GrandSon from '../components/GrandSon.vue'
export default {
  components:{
    GrandSon
  }
}
</script>

////////////////////////孙子组件代码
<template>
  <div class="grandSon">
    我是GrandSon
    {{ color }} -{{ userInfo.name }} -{{ userInfo.age }}
  </div>
</template>

<script>
export default {
  inject: ['color', 'userInfo'],
}
</script>

<style>
.grandSon {
  border: 3px solid #000;
  border-radius: 6px;
  margin: 10px;
  height: 100px;
}
</style>

# 5、ref、$refs，和nextTick

<template>
  <div class="app">
    <div v-if="isShowEdit">
      <input type="text" v-model="editValue" ref="inp" />
      <button>确认</button>
    </div>
    <div v-else>
      <span>{{ title }}</span>
      <button @click="editFn">编辑</button>
    </div>
  </div>
</template>

<script>
export default {
  data() {
    return {
      title: '大标题',
      isShowEdit: false,
      editValue: '',
    }
  },
  methods: {
    editFn() {
      // 1.显示文本框
      this.isShowEdit = true
      // 2.让文本框聚焦 （会等dom更新完之后 立马执行nextTick中的回调函数）
       this.$nextTick(() => {
         console.log(this.$refs.inp)
         this.$refs.inp.focus()
       }) 
    },
  },
}
</script>
