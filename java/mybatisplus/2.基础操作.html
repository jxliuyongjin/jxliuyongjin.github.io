# 一、mybatisplus的增删改查

# 1、增加

User user = new User();
user.setAge(12);
user.setEmail("123@qq.com");
user.setId(123);
user.setName("jxdsdf");
userMapper.insert(user);

# 2、删除

主要说一下deleteByMap，是个条件删除，下列例子就是删除age为12的或者name为jxdsdf的记录，只要满足起中一个条件就删除

Map<String,Object> map = new HashMap<String,Object>();
map.put("age",12);
map.put("name","jxdsdf");
int result = userMapper.deleteByMap(map);

​ 执行的语句是：

Preparing: DELETE FROM user WHERE name = ? AND age = ?

至于其他的deleteById，deleteBatchIds,delet之类的删除函数就没必要记录了

# 3、改数据

User user = new User();
user.setId(1);
user.setEmail("12456@qq.com");
int result = userMapper.updateById(user);
System.out.println(result);

执行的sql语句是：

UPDATE user SET email=? WHERE id=?
Parameters: 12456@qq.com(String), 1(Long)

笔记：从sql语句可以看出，这个只修改有注册属性的数据，没有填写属性值是不会修改的

# 4、查询

List<Long> list = Arrays.asList(1l,2l,3l);
List<User> result = userMapper.selectBatchIds(list);
System.out.println(result);

这是个根据id列表批量查询。还有个selectList(args),这里传递的是一个条件构造器，以后再说，如果args是一个null值，就是查询全部数据。

# 5、自定义查询

# 5.1、添加映射文件

​ 在resources目录下添加mapper文件和xml文件映射文件

​ 内容：

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd" >
<mapper namespace="com.example.mybatisplus.mapper.UserMapper">

    <select id="selectMapById" resultType="map">
        select id,name,age,email from user where id=#{id}
    </select>
</mapper>

​

# 5.2、UserMapper接口添加方法
Map<String,Object> selectMapById(Long id);
# 5.3、测试
 Map<String, Object> users = userMapper.selectMapById(1L);
 System.out.println(users);

​ 执行的sql语句就是自定义的语句了：

Preparing: select id,name,age,email from user where id=?

# 三、注解

# 1、@TableName

当表名和定义的实体类名不一致的时候，可以用@TableName来给实体类指定数据库的表明，使之匹配

@Data
@TableName("t_user")
public class User {
     long id;
     String name;
     Integer age;
     String email;
}

执行的sql语句是

Preparing: SELECT id,name,age,email FROM t_user

当需要全局加一个开头的时候，就可以在properties或者yml里面配置

mybatis-plus: 
  global-config:
    db-config:
      table-prefix: t_
# 2、@TableId
@Data
public class User {
     @TableId("id")
     long uid;
     String name;
     Integer age;
     String email;
}

执行的sql语句是

 SELECT id AS uid,name,age,email FROM user WHERE id=?

如果需要设置自动递增可以设置第二个参数

@TableId(value="id",type = IdType.NONE)

当然数据需要设置自动递增，如果没有设置自动递增是不会其作用的

# 3、通过全局配置主键生成策略
global-config:
  db-config:
    id-type: auto
# 4、@TableField

mybatisplus自动默认驼峰转_的算法

@Data 
public class User {
     long id;
     String userName;
     Integer age;
     String email;
}

起中userName自动匹配表种user_name，调用

 User user = new User();
 user.setUserName("xxxxx");
 userMapper.insert(user);

执行的sql语句是

INSERT INTO t_user ( id, user_name ) VALUES ( ?, ? )
# 5、@TableFiled的用法
@Data 
public class User {
     long id;
     @TableField("user_name")
     String userName;
     Integer age;
     String email;
}

直接直顶对线属性和表种字段的映射关系

# 6、@TableLogic逻辑删除

主要是在表哥种添加一个字段。来标识该条数据是否已经删除

表格种添加is_delete字段并且User对象也添加字段isDelete和@TableLogic注解

@Data
@TableName("t_user")
public class User {
     long id;
     String userName;
     Integer age;
     String email;
     @TableLogic
     Integer isDeleted;
}

执行测试

 int result = userMapper.deleteById(1);
 System.out.println(result);

最后执行的sql语句是：

 UPDATE t_user SET is_deleted=1 WHERE id=? AND is_deleted=0

执行的结果就是is_deleted从0变成了1表示逻辑删除

以后的查找也会自动添加 AND is_deleted=0条件，自动屏蔽is_deleted=1的数据，从而达到逻辑删除的目的

# 四、枚举

# 1、创建enums文件夹和其下面创建SexEnum枚举类
@Getter
public enum SexEnum
{
    MAIL(1,"男"),
    FEMAIL(2,"女");

    @EnumValue //将此注解的值作为值传给数据库
    private Integer sex;
  
    private String sexName;

    SexEnum(Integer sex, String sexName) {
        this.sex = sex;
        this.sexName = sexName;
    }
}

注：a、由于这是一个枚举类，所以只加了一个@Getter注解

​ b、@EnumValue //将此注解的值作为值传给数据库不然会报错

# 2、添加通用枚举扫描

在application.yml中添加配置，

type-enums-package: com.example.mybatisplus.enums
# 3、测试：
User usr = new User();
usr.setUserName("admin");
usr.setAge(33);
usr.setEmail("134@qq.com");
usr.setSex(SexEnum.MAIL);
int result = userMapper.insert(usr);
System.out.println("result:"+result);

执行的sql:

==>  Preparing: INSERT INTO t_user ( id, user_name, age, email, sex ) VALUES ( ?, ?, ?, ?, ? )
==> Parameters: 0(Long), admin(String), 33(Integer), 134@qq.com(String), 1(Integer)

通过sql可以看到是向sex传递了1，从而实现了枚举的应用
